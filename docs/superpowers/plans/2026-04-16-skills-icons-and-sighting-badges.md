## Task 2 — Render Skill icons in `SkillsScreen`

**Files:**

**Data source reminder:** `PluginHome.icon?: string | null` already exists on the summary (`mobile/src/types.ts:22`). Yardsailing's manifest declares `"icon": "storefront-outline"`. No other changes required to get the icon name to this screen.

- [ ] **Step 1: Add the Ionicons import**

At the top of `mobile/src/screens/SkillsScreen.tsx`, after the React Native imports, add:

```tsx

```

(Importing the specific glyph set is cheaper than the whole `@expo/vector-icons` barrel — Metro tree-shakes the rest.)

- [ ] **Step 2: Render the icon in each row**

Inside the `renderItem` callback, replace the current `<Pressable ...>` body with an icon + text column. The diff is: add one `<Ionicons>` before the `<View style={{ flex: 1 }}>`.

Final shape:

```tsx
renderItem={({ item }) => (
  <Pressable
    style={styles.row}
    onPress={() =>
      setSelected({
        pluginName: item.name,
        componentName: item.home.component,
        label: item.home.label,
      })
    }
  >
    <Ionicons
      // Cast is safe: Ionicons accepts any glyph name and warns in dev if
      // unknown. PluginHome.icon is a free-form string from plugin.json.
      name={(item.home.icon ?? "apps-outline") as React.ComponentProps<typeof Ionicons>["name"]}
      size={28}
      color="#475569"
      style={styles.icon}
    />
    <View style={{ flex: 1 }}>
      <Text style={styles.rowTitle}>{item.home.label}</Text>
      <Text style={styles.rowDesc} numberOfLines={2}>
        {item.home.description ?? item.description}
      </Text>
    </View>
    <Text style={styles.chev}>›</Text>
  </Pressable>
)}
```

In the `StyleSheet.create({...})` block, add an `icon` entry after `chev`:

```ts

```

The 12px right margin matches the existing 12px separation elsewhere in the row (padding values on `styles.row` and `styles.list`).

## Task 3 — Render Confirmed/Unconfirmed badge on sighting cards

**Files:**
- Modify: `mobile/src/chat/DataCard.tsx`

**Badge visual spec (lifted from `mobile/src/core/SightingPopup.tsx:79-88`):**
- Unconfirmed bg: `#fef3c7`, label: "Unconfirmed"
- Confirmed bg: `#dcfce7`, label: "Confirmed"
- Text: `fontSize: 11, fontWeight: "700", color: "#0f172a"`
- Container: `paddingHorizontal: 8, paddingVertical: 3, borderRadius: 10`

Threshold: confirmed when `(sale.confirmations ?? 1) >= 2`, matching `SightingPopup`.

- [ ] **Step 1: Add the badge render inside the map card**

In `mobile/src/chat/DataCard.tsx`, inside the `sales.map((sale) => { ... })` render block, find the `<View style={styles.cardInnerRow}>`. Prepend a badge before the `<Text style={styles.title}>`:

Final `cardInnerRow` content:

```tsx
<View style={styles.cardInnerRow}>
  {sale.source === "sighting" ? (
    <View
      style={[
        styles.badge,
        (sale.confirmations ?? 1) >= 2
          ? styles.badgeConfirmed
          : styles.badgeUnconfirmed,
      ]}
    >
      <Text style={styles.badgeText}>
        {(sale.confirmations ?? 1) >= 2 ? "Confirmed" : "Unconfirmed"}
      </Text>
    </View>
  ) : null}
  <Text style={styles.title} numberOfLines={1}>{sale.title}</Text>
  {sale.distance_miles != null ? (
    <Text style={styles.distance}>{sale.distance_miles.toFixed(1)} mi</Text>
  ) : null}
</View>
```

- [ ] **Step 2: Add badge styles**

In the `StyleSheet.create({...})` block at the bottom of the file, add four entries (place them after `title`/`distance` for locality):

```ts
badge: {
  paddingHorizontal: 8,
  paddingVertical: 3,
  borderRadius: 10,
  marginRight: 6,
  flexShrink: 0,
  alignSelf: "flex-start",
},
badgeUnconfirmed: { backgroundColor: "#fef3c7" },
badgeConfirmed: { backgroundColor: "#dcfce7" },
badgeText: {
  fontSize: 11,
  fontWeight: "700",
  color: "#0f172a",
},
```

Rationale for `flexShrink: 0` and `alignSelf: "flex-start"`: the parent `cardInnerRow` is a row with `justifyContent: "space-between"` and `alignItems: "center"`. We want the badge to never truncate when the title grows, and to sit vertically centered without stretching.

## Risks / Notes

- **Parallel session conflict.** Another Claude Code session is working on the 401/404 Create-sale bug in the same repo on branch `fix/skills-yardsailing-create-auth`. That session owns `SaleForm.tsx`, the bridge, and backend routes. We only touch `SkillsScreen.tsx` and `DataCard.tsx` — no overlap expected. Merge-order-wise, land whichever PR is green first; the other should rebase cleanly.
- **`expo-status-bar` install artifact.** On fresh `npm install` in a new worktree, the package's `build/*.d.ts` files are missing, breaking tsc. Task 1 Step 4 documents the one-time copy-from-main workaround. This is a repo-wide environmental quirk, not something this PR should try to fix.
- **Icon name safety.** The `name` prop on `Ionicons` is a TypeScript string-literal union. We cast `item.home.icon` to that union to avoid changing the plugin-manifest contract (`icon: string | null`). Runtime safety comes from Ionicons itself (warns on unknown, renders empty). If in a future PR the plugin registry validates icon names at registration time, the cast can be removed.

## Non-goals

- No icon-per-plugin asset pipeline (PNG/SVG shipped in the bundle) — defer until a plugin actually needs it.
- No filter chip bar in chat-map card — user explicitly chose option B over option A.
- No `SaleDetailsModal` or `MySales`-legacy-view changes.
- No tests added — follows the established pattern for mobile UI-only changes in this repo.
